monkey是Android自动化测试工具

monkey主要应用在压力和可靠性测试上，业务流程的测试还是需要依靠其他方法

##开始Monkey

这个monkey 命令，当monkey test 过程中遇到Crash 或者ANR，就会自动终止.
>
那么需要执行多久呢，100次，176ms，这个时间并不是100*20ms

  • throttle 20:每一个操作的间隔 380ms
  • s 3500:类似于随机数种子，同一个值会产生一个固定操作序列
  • 300000：操作序列中的操作数目
  • v:日志等级，对应还有-v -v、 -v -v -v
  • “> D:\monkey_log.txt”：日志保存的位置

结束monkey

  1. ps命令 查找uiautomator的进程
    打开cmd命令行窗口
    输入：
    adb shell ps | grep monkey
    返回来的第一个数字，即是monkey的进程号(如果提示grep找不到，则使用adb shell ”ps | grep monkey“)
  2. kill 命令结束进程继续输入:
    adb shell kill [刚才查到的进程号]
    同理结束其他进程，和其他应用也可用此法，屡试不爽
  3. ctrl +c结束无效，但是会结束日志文件的写入

注意

  1. 会触发哪些操作
    触摸、音量，截屏、飞行模式等都有(有时候竟然还开启了音乐，没注意到怎么开启的)
  2. 查看触摸轨迹
    可以开启adb调试中的显示触摸来查看该触摸事件的触摸轨迹
  3. 是在正式环境还是测试环境上测试
    得看具体的软件，如果有上报等功能，建议还是测试环境，不然，一不小心上报给谁了，就尴尬了，毕竟这个测试是自动的，不是手动控制的
  4. 如果有webView和url连接会怎样
    如果点进去了，也会随机触摸，当然，最终还是会触发返回
  5. 如果有输入框会怎样
    也会随机触发输入，滑动、音量等，但是，如果需要手动点完成(毕竟这才是正常流程)，那不一定能成功，毕竟随机触发(偶尔触发，偶尔不触发)
  6. 如果一不小心monkey了退出登录，那么，接下来的monkey，就只能在登录界面了
  7. 关于测试效率
    合理调整事件百分比，默认情况下，经常进入的测试界面就那几个，所以覆盖面不广
  8. –pct-rotation 0 就算设置了这一个，触发的最后一个事件，还是会打开rotation
    发送事件总数为 1， 自动旋转被打开(原先是关闭状态)
    Events injected: 1
    :Sending rotation degree=0, persist=false
    :Dropped: keys=0 pointers=0 trackballs=0 flips=0 rotations=0
    Network stats: elapsed time=31ms (0ms mobile, 0ms wifi, 31ms not connected)
    // Monkey finished
  9. monkey之后，如果出现了一些奇怪的问题(比如输入法突然没有自动提示)，别感到奇怪，毕竟是随机触发，说不定被点中了什么
  10. 日志中有一个switch是什么
    没看懂，在我的例子中，一直都是LoadActivity
  11. 日志中前面的几个百分比是什么(不同sdk中顺序可能不一样)
    // Event percentages:
    // 0: 30.0% // touch触摸事件的百分比(触摸事件是一个down-up事件，它发生在屏幕上的某单一位置)
    // 1: 8.235294% // motion 动作事件的百分比(动作事件由屏幕上某处的一个down事件、一系列的伪随机事件和一个up事件组成)直线滑动
    // 2: 1.6470588%
    // 3: 12.3529415% //trackball 轨迹事件的百分比(轨迹事件由一个或几个随机的移动组成，有时还伴随有点击)曲线移动
    // 4: -0.0%
    // 5: 20.588236% // nav “基本”导航事件的百分比(导航事件由来自方向输入设备的up/down/left/right组成)
    // 6: 12.3529415% //majornav “主要”导航事件的百分比(这些导航事件通常引发图形界面中的动作，如：5-way键盘的中间按键、回退按键、菜单按键)
    // 7: 1.6470588% //syskeys “系统”按键事件的百分比(这些按键通常被保留，由系统使用，如Home、Back、Start Call、End Call及音量控制键)
    // 8: 1.6470588% //appswitch 启动Activity的百分比。在随机间隔里，Monkey将执行一个startActivity()调用，作为最大程度覆盖包中全部Activity的一种方法
    // 9: 0.8235294%
    // 10: 10.705882% //anyevent 其它类型事件的百分比。它包罗了所有其它类型的事件，如：按键、其它不常用的设备按钮、等等
